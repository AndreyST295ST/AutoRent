Модель предметной области
Сервис по аренде автомобилей
Модель предметной области описывает ключечные сущности сервиса по аренде автомобилей, а также взаимосвязи между ними. Модель отражает основные бизнес-объекты, используемые в системе, и служит основой для формирования функциональных и нефункциональных требований.

Пользователи системы
Базовой сущностью является Пользователь (User), представляющая зарегистрированного пользователя сервиса. Пользователь характеризуется уникальным идентификатором, именем, фамилией, электронной почтой, номером телефона, хешем пароля, датой регистрации и статусом верификации.

В системе выделяются три специализированные роли пользователей:

Клиент (Client) — арендатор, осуществляющий поиск, бронирование и аренду автомобилей, оставляющий отзывы;

Владелец автопарка (CarOwner) — партнёр сервиса, добавляющий автомобили в систему и управляющий своим автопарком;

Администратор (Administrator) — сотрудник сервиса, осуществляющий модерацию контента, управление пользователями и контроль работы системы.

Связь между сущностями реализована с использованием наследования: каждая роль является специализированным видом пользователя.

Для подтверждения личности и права управления транспортным средством используется сущность Водительские права (DriverLicense). Она содержит номер документа, даты выдачи и окончания срока действия, категории транспортных средств, а также фотографии документа. Водительские права принадлежат конкретному пользователю и проходят процедуру верификации.

Для осуществления платежей используется сущность Способ оплаты (PaymentMethod), представляющая банковскую карту пользователя. Сущность хранит токен карты (без сохранения реальных данных), последние четыре цифры, имя владельца и срок действия. Пользователь может иметь несколько способов оплаты, один из которых может быть основным.

Автомобили и автопарк
Сущность Автомобиль (Car) описывает транспортное средство, доступное для аренды. Автомобиль содержит идентификатор, марку, модель, год выпуска, государственный номер, цвет, тип коробки передач, тип топлива, количество мест, фотографии и текущие координаты местоположения.

Для хранения детальных технических характеристик используется сущность Характеристики автомобиля (CarSpecs), которая включает мощность двигателя, объём, расход топлива, объём багажника, а также наличие дополнительных опций (кондиционер, подогрев сидений, навигатор, Bluetooth). Характеристики связаны с конкретным автомобилем отношением один-к-одному.

Сущность Автопарк (Fleet) представляет собой совокупность автомобилей, принадлежащих одному владельцу. Автопарк содержит название и общее количество машин. Владелец автопарка может управлять несколькими автомобилями, которые входят в его автопарк (агрегация).

Бронирование и аренда
Сущность Бронь (Booking) является центральной в процессе аренды. Она создаётся клиентом и связывает его с выбранным автомобилем. Бронь содержит уникальный номер, планируемое время начала и окончания аренды, фактическое время начала и окончания, итоговую стоимость и статус (ожидание, подтверждена, активна, завершена, отменена).

В процессе аренды порождается сущность Сессия аренды (RentalSession), которая фиксирует детали конкретной поездки: показания пробега на начало и конец, уровень топлива, фактическое время начала и окончания, общее время в минутах и пройденное расстояние. Сессия аренды связана с бронированием отношением один-к-одному.

Для учёта нештатных ситуаций используется сущность Инцидент (Incident), которая описывает происшествия во время аренды: ДТП, поломку, штраф или просрочку возврата. Инцидент содержит тип, описание, фотографии, дату сообщения и дату решения. Инцидент может быть связан с конкретной сессией аренды.

На основе инцидентов может создаваться сущность Штраф (Penalty), содержащая сумму штрафа, причину начисления, дату и статус оплаты.

Платежи и финансы
Сущность Платеж (Payment) отражает все финансовые операции, связанные с арендой. Она включает сумму, заблокированную сумму (залог), итоговую сумму к списанию, статус платежа (ожидание, заблокировано, оплачено, возвращено, ошибка), идентификатор транзакции во внешней платёжной системе и дату платежа. Одна бронь может включать несколько платежей (например, предварительная блокировка и финальное списание).

Для формирования отчётных документов используется сущность Чек (Invoice), которая содержит уникальный номер, дату выдачи, позиции чека, сумму без налога, налог и итоговую сумму. Чек формируется на основе бронирования и отправляется клиенту.

Сущность Позиция чека (InvoiceItem) описывает отдельную строку в чеке: описание услуги, количество, цену за единицу и сумму. Чек содержит несколько позиций (композиция).

Для аудита всех финансовых операций используется сущность Лог транзакций (TransactionLog), которая записывает каждое действие с платежом (создание, блокировка, списание, возврат) с указанием времени и деталей. Логи существуют только в контексте конкретного платежа (композиция).

Отзывы и обратная связь
Сущность Отзыв (Review) представляет текстовый отзыв клиента об арендованном автомобиле. Отзыв содержит числовую оценку (от 1 до 5), текстовый комментарий и дату создания. Отзыв связан с конкретным клиентом и конкретным автомобилем. Клиент может оставить только один отзыв на каждый автомобиль после завершения аренды.

Уведомления
Сущность Уведомление (Notification) используется для информирования пользователей о событиях в системе: подтверждение брони, напоминание о предстоящей аренде, окончание аренды, новые предложения, статус верификации и т.д. Уведомление содержит тип, заголовок, сообщение, дату отправки и флаг прочитано/не прочитано. Уведомления адресуются конкретному пользователю.

Управление и аналитика
Сущность Отчет (Report) используется администраторами и владельцами автопарков для анализа работы сервиса. Отчет содержит тип (загрузка автопарка, доходы, популярные автомобили), период, дату генерации и данные в структурированном виде. Отчеты создаются администраторами системы.

Сущность Администратор (Administrator) обладает расширенными правами для управления системой: модерация пользователей, проверка документов, управление автомобилями, просмотр отчетов, решение инцидентов.

Сущность Владелец автопарка (CarOwner) представляет партнёра сервиса. Помимо стандартных полей пользователя, содержит название компании и ИНН. Владелец может добавлять автомобили в свой автопарк, просматривать статистику по своим автомобилям и выводить заработанные средства.
